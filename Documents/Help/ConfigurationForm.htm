Configuration

By using configuration form, you can configure overall settings of SpiderU, such as outpu file format, automatic file name option, &c.

Choose file format

There are three output file formats supported:

  • CSV(Comma Separated Value) file
  • HDF5(Hierarchical Data File)
  • LD1(Locally defined Data file)

Detail of output file format will be found here

Auto file name

SpiderU can generate output file name automatically. Auto file name consits of prefix, serial number and suffix. Auto file name prefix can be generated according to current date and time from meta character, specified by auto file name initial value. Format of meta character is the same as .NET framework's data and time custom format string.

You can learn more at MSDN: Custom Date and Time Format Strings

For example, yyyyMMdd will be converted into something like 20141005.
Arbitrary string can be include in initial value by quoting in double quotes. For example, when you are collecting data about VISITORS from other planets, you can set auto file name initial value like
"Roswell-Greys-"MMM-dd-yyyy-
This meta string would be converted into, say,
Roswell-Greys-Sep-19-1961-

CSV file option

CSV file can include comment line and header line. Comment line begins by "#" character. Character encoding can be selected from

  • ASCII
  • UTF-8
  • Unicode
Detailed information will be found at File format.

Synchronize

SpiderU can treat multiple number of DSOs as one many channel DSO. Using this option checked, SpiderU assumes that all DSOs are triggered by same trigger source and all DSO channes has same record length, same sampling time. Plaease make sure that YOU have to set DSOs synchoronized. Especially, triggering synchronization may be not that easy. Theoretically, synchronization would be accomplished not by the same model of DSOs. But, in practical case, using same model of DSOs will make it easy.
